## 一、产品概述

### 1.1 产品定位

「门外审堂」读书学习系统是一款**以读书计划驱动、以每日反思为核心、以AI导师互动为辅助**的沉浸式学习工具。帮助堂主通过系统化的交互流程，深度完成《门外审堂》的阅读学习，并将每日所学凝练为"三件事"反馈，与文渊先生（AI导师）进行深度交流。

### 1.2 目标用户

| 角色 | 说明 |
|------|------|
| **堂主（主要用户）** | 使用系统完成读书计划、提交每日反馈、与文渊先生互动 |
| **文渊先生（AI导师）** | 负责内容质量把关、读书反馈点评、深度交流指导 |
| **软件产品堂堂主** | 产品设计与迭代管理 |
| **码农程序猿** | 系统开发与维护 |

### 1.3 平台要求

- ✅ **Windows** — 桌面端应用
- ✅ **iOS** — iPhone/iPad 应用
- ✅ 两端数据实时同步

### 2.2 模块详细说明

#### 📖 模块一：读书计划管理

| 功能 | 说明 |
|------|------|
| **计划总览** | 展示《门外审堂》全书结构、章节安排、预计完成时间 |
| **每日任务** | 根据计划自动生成当天阅读任务（章节/页码/时长） |
| **进度追踪** | 可视化进度条，已完成/进行中/未开始章节标识 |
| **阅读笔记** | 每章可附带个人笔记、高亮摘录、思维导图 |
| **打卡签到** | 每日阅读完成打卡，记录连续学习天数 |

#### ✍️ 模块二：每日三件事反馈

| 功能 | 说明 |
|------|------|
| **三件事模板** | 每日阅读后填写：①今日学到了什么 ②最大的触动/思考 ③准备如何实践 |
| **提交反馈** | 一键提交给文渊先生，进入导师点评队列 |
| **历史回顾** | 按日期/章节查看所有历史反馈记录 |
| **导师批注** | 文渊先生对每条反馈的点评与指导 |
| **成长轨迹** | 可视化展示认知成长曲线与反思深度变化 |

#### 💬 模块三：与文渊先生交流

| 功能 | 说明 |
|------|------|
| **即时对话** | 与文渊先生进行实时文字交流（基于AI对话） |
| **主题讨论** | 围绕书中章节/观点发起专题讨论 |
| **问答解惑** | 针对阅读中的疑问向文渊先生提问 |
| **读书报告** | 阶段性生成读书报告，由文渊先生评审 |
| **交流记录** | 所有对话历史可搜索、可归档 |

#### 🧠 模块四：知识体系管理

| 功能 | 说明 |
|------|------|
| **知识图谱** | 将书中核心概念以图谱形式可视化呈现 |
| **概念卡片** | 关键概念的结构化卡片（定义/关联/案例） |
| **主题阅读路径** | 文渊先生规划的延伸阅读路径 |
| **知识关联** | 将不同章节的知识点进行关联标注 |

#### 👤 模块五：个人中心

| 功能 | 说明 |
|------|------|
| **学习统计** | 总阅读时长、完成章节数、连续打卡天数 |
| **成就系统** | 学习里程碑徽章（如"连续7天打卡""完成第一章"） |
| **数据导出** | 导出读书笔记、三件事记录为PDF/Markdown |
| **系统设置** | 提醒时间、主题切换、账号管理 |

## 三、技术架构设计

### 3.1 技术选型

| 层级 | 技术方案 | 选择理由 |
|------|---------|---------|
| **跨平台框架** | **Tauri 2.0** | 轻量级、原生性能好、支持Windows原生+iOS，包体小 |
| **前端框架** | **React 18 + TypeScript** | 生态成熟、组件丰富、类型安全 |
| **UI 组件库** | **Ant Design 5.x** | 企业级UI组件、中文支持好 |
| **状态管理** | **Zustand** | 轻量、简洁、性能好 |
| **后端服务** | **Node.js + Express** | 前后端统一语言、快速开发 |
| **数据库** | **SQLite (本地) + Supabase (云端)** | 本地优先+云同步，离线可用 |
| **AI 对话** | **Coze API 集成** | 对接文渊先生的AI能力 |
| **实时通信** | **WebSocket** | 支持即时对话与消息推送 |

### 3.2 为什么选 Tauri 而不是 Flutter/Electron？

| 对比项 | Tauri 2.0 | Electron | Flutter |
|--------|-----------|----------|---------|
| Windows 支持 | ✅ 原生WebView2 | ✅ Chromium | ✅ 自渲染 |
| iOS 支持 | ✅ 原生WKWebView | ❌ 不支持 | ✅ 自渲染 |
| 包体大小 | ~5MB | ~150MB | ~30MB |
| 内存占用 | 低 | 高 | 中 |
| 开发语言 | Rust + Web前端 | JS | Dart |
| 学习成本 | 低（Web技术栈） | 最低 | 中（需学Dart） |

**结论**：Tauri 2.0 在包体大小、性能、跨平台覆盖度上综合最优，且团队已有Web前端能力，无需额外学习新语言。

### 3.4 数据模型设计

#### 核心数据表

```sql
-- 读书计划
CREATE TABLE reading_plan (
  id            TEXT PRIMARY KEY,
  book_title    TEXT NOT NULL,          -- 《门外审堂》
  total_chapters INTEGER,
  start_date    DATE,
  end_date      DATE,
  status        TEXT DEFAULT 'active',  -- active | completed | paused
  created_at    TIMESTAMP DEFAULT NOW
);

-- 章节任务
CREATE TABLE chapter_task (
  id            TEXT PRIMARY KEY,
  plan_id       TEXT REFERENCES reading_plan(id),
  chapter_num   INTEGER NOT NULL,
  chapter_title TEXT,
  start_page    INTEGER,
  end_page      INTEGER,
  due_date      DATE,
  status        TEXT DEFAULT 'pending', -- pending | in_progress | completed
  completed_at  TIMESTAMP,
  notes         TEXT                    -- 个人笔记
);

-- 每日三件事
CREATE TABLE daily_three (
  id            TEXT PRIMARY KEY,
  date          DATE NOT NULL,
  learned       TEXT NOT NULL,          -- ① 今日学到了什么
  reflection    TEXT NOT NULL,          -- ② 最大的触动/思考
  action        TEXT NOT NULL,          -- ③ 准备如何实践
  chapter_id    TEXT REFERENCES chapter_task(id),
  status        TEXT DEFAULT 'draft',   -- draft | submitted | reviewed
  mentor_reply  TEXT,                   -- 文渊先生的点评
  submitted_at  TIMESTAMP,
  reviewed_at   TIMESTAMP
);

-- 对话消息
CREATE TABLE conversation (
  id            TEXT PRIMARY KEY,
  topic         TEXT,                   -- 讨论主题
  role          TEXT NOT NULL,          -- user | mentor
  content       TEXT NOT NULL,
  chapter_id    TEXT,                   -- 关联章节（可选）
  created_at    TIMESTAMP DEFAULT NOW
);

-- 知识卡片
CREATE TABLE knowledge_card (
  id            TEXT PRIMARY KEY,
  concept       TEXT NOT NULL,          -- 概念名称
  definition    TEXT,
  chapter_ids   TEXT,                   -- 关联章节 (JSON array)
  related_ids   TEXT,                   -- 关联卡片 (JSON array)
  created_by    TEXT                    -- user | mentor
);

-- 学习统计
CREATE TABLE learning_stats (
  id            TEXT PRIMARY KEY,
  date          DATE NOT NULL UNIQUE,
  reading_mins  INTEGER DEFAULT 0,     -- 阅读时长(分钟)
  pages_read    INTEGER DEFAULT 0,
  checked_in    BOOLEAN DEFAULT FALSE, -- 是否打卡
  streak_days   INTEGER DEFAULT 0      -- 连续天数
);
```

## 四、核心交互流程

### 4.1 每日学习主流程

```
堂主打开应用
    │
    ▼
查看今日阅读任务 ←── 系统根据读书计划自动生成
    │
    ▼
开始阅读（计时开始）
    │
    ├── 记录笔记/高亮
    │
    ▼
完成阅读 → 打卡签到
    │
    ▼
填写"每日三件事"
    │
    ├── ① 今日学到了什么
    ├── ② 最大的触动/思考
    └── ③ 准备如何实践
    │
    ▼
提交给文渊先生 ←── 通过Coze API推送
    │
    ▼
文渊先生点评反馈 ←── 异步返回
    │
    ▼
堂主查看点评 / 发起深入讨论
```

### 4.2 与文渊先生交流流程

```
堂主发起对话
    │
    ├── 选择方式
    │   ├── 自由提问（开放式对话）
    │   ├── 章节讨论（关联特定章节）
    │   └── 读书报告（阶段性总结请求）
    │
    ▼
进入对话界面（实时聊天）
    │
    ▼
文渊先生回复（AI生成 + 内容质量把控）
    │
    ├── 可追问深入
    ├── 可保存为笔记
    └── 可生成知识卡片
```

## 五、跨平台技术方案

### 5.1 统一代码库策略

```
src/                    # 共享代码（95%+）
├── components/         # React UI 组件
├── pages/              # 页面组件
├── stores/             # Zustand 状态管理
├── services/           # API 调用层
├── hooks/              # 自定义 Hooks
├── utils/              # 工具函数
├── types/              # TypeScript 类型定义
└── styles/             # 全局样式

src-tauri/              # Tauri 原生层
├── src/
│   └── main.rs         # Rust 入口
├── capabilities/       # 权限配置
└── Cargo.toml

# 平台特定配置
├── src-tauri/          # Windows 构建配置
└── gen/schemas/        # iOS 构建配置
```

### 5.2 平台适配要点

| 适配项 | Windows | iOS |
|--------|---------|-----|
| 导航方式 | 侧边栏导航 | 底部Tab导航 |
| 字体大小 | 默认14px | 自适应（支持Dynamic Type） |
| 交互方式 | 鼠标hover + 点击 | 触摸手势 |
| 离线支持 | SQLite 本地存储 | SQLite 本地存储 |
| 推送通知 | Windows Toast | APNs |
| 文件导出 | 直接保存到本地 | 分享面板 |

### 5.3 离线优先策略

```
用户操作 → 写入本地 SQLite → 加入同步队列
                                    │
                              检测网络状态
                              ├── 有网 → 立即同步到云端
                              └── 无网 → 队列等待，联网后批量同步
                                    │
                              冲突处理（CRDT）
                              └── 以时间戳 + 字段级合并
```
